TypeScript Introduction:

TypeScript is an Open-Source-Project developed by Microsoft. It expands JavaScript, which means that you can write as much JavaScript in your TypeScript document as you want.

But it comes with a much stronger type system, as you may know from programing languages like Java and C#.

Bonus: TypeScript is runnable in every browser and platform as it compiles to pure JavaScript.

1. Basic Types

When you declare and initialise a variable, it's always the same:

name: type = value;
Number
age: number = 5;
String
firstname: string = "Max Mustermann";

Template-String: Expressions embedded directly in the string.

output: string = "${firstname} is ${age} years old.";
Getter and Setter

Known from object oriented programing. You can read or set the property of an object.

Boolean
let workingHard: boolean = true;
Arrays
let fibonacci: Array<number> = [0, 1, 1, 2, 3, 5, 8, 13];
let fibonacci: number[] = [0, 1, 1, 2, 3, 5, 8, ,13];
let differentValueTypes: any[] = [5, { firstname: "Max Mustermann" }, function() {
		console.warn("Nothing inside.")
		}];
Any

You can always use the fallback value "any" if you don't know which data type you will get back.

2. Classes

class Animal { }
Properties

Extend classes with further information. An animal could have name, age and owner.

class Animal { 
    public name: string;
    public age: number;
    public owner: string;
}
Methods

Extend classes with logic. In addition TypeScript allows to precise the signature of the methods with a parameter type and a return type. 

class Animal { 
    public name: string;
    public age: number;
    public owner: string;

    changeOwner(owner: string): string {
        this.owner = owner;
        return this.owner;
    }
}

If you use "void" as return type, the method has no return value.

Constructors

Constructors are methods which are called during the initialization of a class. Values that are necessary later, can be transferred.

class Animal {
    public name: string;
    public age: number;
    public owner: string;

    constructor(name: string, age: number, owner: string) {
        this.name = name;
        this.age = age;
        this.owner = owner;
    }
}

As shorthand the use of public on constructor arguments automatically creates these properties:

class Animal {
    constructor(public name: string, public age: number, public owner: string) {
    }
}

A difference to other programing languages: Only ONE constructor per class is possible!

Inheritance

Also known from object-oriented programing, with the keyword extends.

class Cat extends Animal { }

You can use the methods and constructors of the base class with super(). The methods can be rewritten. If a derived class (Cat) is instantiated, you automatically get access to attributes and methods of the parent classes (Animal).

3. Interfaces

Interfaces are used to define a public interface and hide the concrete implementation. It can be determined which parts must be included and which types they should have.

Optional parameters have a ? behind their name e.g. public owner?: string;.

interface Contact {
    firstname: string;
    lastname: string;
    age?: number | string;
}

If the object gets an additional property or the value doesn't have the required type, you would get an error.

Indexable Interfaces

Interfaces can also be defined by an indexable key. For instance the interface determines the type for all elements of an array.

interface StringArray {
    [index: number]: string;
}
Interfaces for Functions

The key is the function signature. The value of the interface is the return value of the function.

Interface for Classes

The keyword to implement an Interface is implements. The compiler will display what the implementation of the interface requires. That means all defined properties and methods of the interface. Type definitions are the same as in interfaces.

4. Decorators

@Component({
    // Configuration object
})
class myComponent { }

Metadata is added to the class. The Angular-Framework knows that the class must be handled as a component.

Expressions such as @readonly are not provided in Angular.
