Components

What are Components?

A component controls a part of the screen. In the HTML you define the structure of a component and the CSS styles this structure, togehter it's the view of the component. The TypeScript file is the logic part of the component. An Angular application is basically an application with a tree structure. It has a root, that is the AppComponent and it's also the entry point of the application. However, there is no difference to the other components in concept or syntax. Only difference is that the Angular framework knows that this component is the entry point of the application. If you have the root component you can put other components inside. You can interlace the components more and more. This is the most important architecture concept of Angular.

Generate new Component

Open your command line in Visual Studio Code and generate (short form g) your "where-to-go" Component, it's going to be the page you see first when you start the app.

ng g component where-to-go

Result:

The app.module.ts automatically imported your new component. The CLI creates three files in one folder for you. It's the skeleton for your component. The HTML file is nearly empty and ready for your code. You can ignore the spec.ts file it defines a unit test for the WhereToGoComponent. The .ts file is for your TypeScript code and defines the components logic.

There are some interesting things about the skeleton of the component.ts files. @Component is a decorator function which specifies the Angular metadata for the component. The CLI generates three metadata properties:

  • selector: the CSS-element-selector of the component
  • templateURL: the location of the components' template file
  • styles: the location of the private CSS style of the component (here empty, because we decided to import the semantic-ui for a overall style and therefor generated the project without a css for every component)

The selector 'app-where-to-go' matches with the name of the HTML element. This name identifies the component within the template of the parent-component.

ngOnInit is a lifecycle hook function. Angular calls this function shortly after creating the component. It is the right function to place initialisation logic.

The component always class needs to be exported, to reuse it anywhere else. E.g. in the AppModule.

Reuse Component

We can now use the selector

<app-where-to-go></app-where-to-go>

and insert it into the app.component.html. Now the app shows the component at this location, just like it would do with every other HTML tag.

Communication between Logic and View

As we already learned every component consists of two parts, logic (means component.ts) and view (means component.html). To ensure a communication between these two we can use different types of bindings.

  1. Property Binding:
    they allow you to set the value of a DOM property based on an expression. You assign them with square brackets.
    <element [propterty]="expression">...</element>
  2. Interpolation:
    helps to display properties of the component or calculated values as strings in the HTML view.
    <div>{ { component-property } }</div>
  3. Event Binding:
    this is kind of the opposite of property binding. It is possible to react to the occurrence of DOM events, in order to change the application model, for example.
    <element (eventname)="action">...</element>
  4. Two-Way Binding:
    use this if you want to bind a input component directly to a value of your component class.
    <input [(ngModel)]="text">...</input>
    It is a combination of two bindings and you could write the expanded form too.
    <input [ngModel]="text" (ngModelChange)="setOtherText($event)">...</input>
  5. Local Template-Variables:
    you can create a local variable. With this you have now access to the whole DOM-API of the input field.
    <element #myElement>...</element>

Create and show Property

Now we will use data-binding to show a temporary text message on the "Home" page until we are ready to show the information about the next lecture we have to go to. Put this property into where-to-go.component.ts:

tempText: string = "Here will appear your information soon!";

and replace the auto generated code in the where-to-go.component.html with this code:

{ { tempText } }

This kind of data-binding is called interpolation and you just displayed the value of the property "tempText" via your HTML code.

Interpolation: Shows data in the template. Syntax:
{ {   } }

Create Class

For our project we are going to need a class to define a lecture. With this code you can generate a TypeScript class in a new folder named "shared":

ng g class shared/lecture

In this new class we define the constructor for lecture.

constructor(
  public name: string,
  public room: string,
  public day: string,
  public time: Time,
  public building: string,
  public floor: string,
  public department?: string
){ }

The property department is optional in this constructor and therefor marked with a ?. Angular has its own type for Time. Usually Visual Studio Code should ask you right away if you want to import it.

If not, you can force it by placing the curser behind the underlined word and press STRG + space bar, or do it on your own like this:

import { Time } from "@angular/common";

Pipes

In every application you will get data, you will transform the data and present it to the user. This data could be just a simple local string variable, but also much more. You could display the data as raw values or you could format them to achieve a better user experience. Angular comes along with many build-in pipes. With pipes you can format strings, numbers and other displayed data. Additionally it is possible to create your own pipes. In this case we are fine with the build-in pipes. For example you can format the date of today. When you instantiate a Date object like this:
today: Date = new Date();
You will get this format when you display it in your HTML just "raw":

{{today}} ---- code: { {today} }

Probably users what to have another format for a date. So you can use pipes in your HTML to format the date like this:

{{today | date}} ---- code: { {today | date} }

{{today | date: 'medium'}} ---- code: { {today | date: 'medium'} }

{{today | date: 'fullDate'}} ---- code: { {today | date: 'fullDate'} }

We want to try the UppercasePipe, which formats our string in capital letters:

<div class="header">What? { {lecture.name | uppercase} } </div>
