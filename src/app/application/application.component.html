Initial Application

Open your cmd and type:

ng new where-to-go -is

The -is defines that the project has an inline-style. If you don't do this, we would get an extra CSS-file for every single component. We don't want that for this project.

Next open your Visual Studio Code (restart it if it was open during the other installations) and open the project folder (STRG+O). After that run the initial application. You can open a terminal directly in Visual Studio Code with STRG+Ã–.

ng serve --open

Angular Modules

Angular modules help structuring the application by function. Technically modules are classes like all other Angular blocks. The NgModule-decorator makes the class a module. Decorators define metadata which Angular will need at runtime to initialise the application.

Decorators are specific functions that can be used for classes and their properties, methods and parameters. They are called at runtime and expand the functionality of a class. Decorators always start with an @-symbol.

The class main.ts is the main Angular module for every project. With declarations all components, directives and pipes that belong to this module are declared. With providers all providers are registered. In exports you specify all components, directives and pipes which should be available for other Angular modules. imports contains all Angular modules on which this module depends. And bootstrap defines the main component of the application. The main component is the root of the application. As you will learn in the chapter Components the root is the AppComponent.

You can use different module types such as the main module (module.ts), routing-module and service-module. A routing-module contains configuration for the router. You will learn about the router in the chapter Router. A service-module provides logic / functions for the entire application as you will learn in the chapter Service.

Make some changes

Open app.component.ts and change the title of the app.

export class AppComponent {
  title = 'Where To Go';
}

In the app.component.html is a binding to "title". This binding brings the new title into the application. 

<h1>Welcome to {{ title }}!</h1>

Import UI-Framework

As this is a web application showing you how to build an Angular app, there is no need to do CSS styling. Probably you already know how to style with CSS and the focus here should be Angular. That's why we are going to use the Semantic UI CSS-Framework for a nice look and feel.

With the + button you can add another terminal to type in new commands while running the application.

You can install the UI-Framework with the command:

npm i semantic-ui-css

Add to the file angular.json:

"styles": [
    "node_modules/semantic-ui-css/semantic.css",
    "src/styles.css"
]

Now you can test the new style. Adjust the app.component.html like this and see if it looks different:

<div style="text-align:center">
  <h1 class="ui header">
    Welcome to {{ title }}!
  </h1>
</div>

If you have any problems with installing the semantic ui css, here are some tips that may help. If you get the message "found 1 high severity vulnerability" on the console, just run "npm audit fix". Next you can try a restart of Visual Studio Code. If this is also not working, you can download it by yourself (Semantic UI CSS Download) and paste it in your project folder.

If the Semantic UI CSS-Framework is not automatically loaded into your web page, try to restart the app with ng serve --open
