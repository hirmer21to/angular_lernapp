App

Initial Application

Open your cmd and type:

ng new where-to-go -is

The -is defines that the project has an inline-style. If you don't do this, we would have an extra CSS-file for every single component. For this project, we don't want this.

Next open your Visual Studio Code and open project from file (STRG + K    STRG+O). And then run the initial application. You can open a terminal directly in Visual Studio Code with STRG+รถ.

ng serve --open

Structure of the project

The CLI created a project for us. These are the workspace files:

Where-To-Go
|- e2e: file for end-to-end tests
|   |-- ...
|- node_modules: installed npm-packages
|   |-- ...
|- src: code of our application
|   |-- ...
|- .editorconfig: configuration for the code editor
|- .gitignore: files and folders ignored by Git
|- angular.json: configuration file of the angular CLI
|- package-lock.json: contains information about installed packages
|- package.json: npm-packages configuration and information
|- README.md: documents help
|- tsconfig.json: configuration file for TypeScript
|- tslint.json: TSLint configuration file

In the src folder is our project code. These are the src files:

src
|- app: contains modules and components
|   |-- ...
|- assets: folder for images and files
|   |-- ...
|- environments: configuration for environment
|   |-- ...
|- browserlist: configuration for the code editor
|- favicon.ico: icon used in the browser bookmark bar
|- index.html: main HTML page
|- main.ts: compiles the application
|- polyfills.ts: polyfill scripts
|- styles.css: stylesheet for the project
|- test.ts: entry point for unit tests
|- tsconfig.app.json: inherits from the workspace-wide tsconfig.json file
|- tsconfig.spec.json: inherits from the workspace-wide tsconfig.json file
|- tslint.json: inherits from the workspace-wide tslint.json file

In the app folder is the logic and data of our application. It always contains the follwing files:

src
|- app-routing.module: provided for the router (navigation)
|- app.component.html: HTML template, shell of the application and called in the index.html with app-root
|- app.component.spec.ts: defines unit test for the root AppComponent
|- app.component.ts: defines logic for the AppComponent
|- app.module.ts: all used modules are declared here

Git

Before we make changes, we want to manage our project with Git. As you already know, Visual Studio Code comes with a Git integration. So my recommendation is to install Sourcetree, open a new tab and add the project folder and manage Git from Sourcetree. Of cause, you can also use Git Gui or Visual Studio Code.

Make some changes

Open app.component.ts and change the title of the app.

export class AppComponent {
   title = 'Where To Go';
}

In the app.component.html is a binding to "title". This binding brings the new title into the application.

Welcome to { { title } }!

Import UI-Framework

As this is a web application showing you how to build an Angular app, there is no need to do CSS styling. Probably you already know how to style with CSS and the focus here should be Angular, that's why we are going to use the Semantic UI CSS-Framework for a nice look and feel.

With the plus button you can add another terminal to type in new commands while running the application.

You can install the UI-Framework with the command:

npm i --save semantic-ui-css

Add to the file angular.json:

"styles": [
  "../Where-To-Go/node_modules/semantic-ui-css/semantic.css",
  "src/styles.css"
],

Now you can test the new style. Adjust the app.component.html like this and see if it looks different:

<div style="text-align:center" >
  <h1 class="ui header" style="margin: 30px">
    Welcome to { { title } }!
  </h1>
</div>
<router-outlet> </router-outlet>

If you have any problems with installing the semantic ui css, here are some tips that may help. If you get on the console the message found 1 high severity vulnerability just run npm audit fix. Next you can try a restart of Visual Studio Code. If this is also not working. You can download it by yourself (Semantic UI CSS Download) and paste it in your project folder.
