App

Initial Application

Open your cmd and type:

ng new where-to-go -is

The -is defines that the project has an inline-style. If you don't do this, we would have an extra CSS-file for every single component. For this project, we don't want this.

Next open your Visual Studio Code and open project from file (STRG + K    STRG+O). And then run the initial application. You can open a terminal directly in Visual Studio Code with STRG+รถ.

ng serve --open

When the brower is open you will see the inital app of Angular:

Git

Before we make changes, we want to manage our project with Git. As you already know, Visual Studio Code comes with a Git integration. So my recommendation is to install Sourcetree, open a new tab and add the project folder and manage Git from Sourcetree. Of cause, you can also use Git Gui or Visual Studio Code.

Make some changes

Open app.component.ts and change the title of the app.

export class AppComponent {
   title = 'Where To Go';
}

In the app.component.html is a binding to "title". This binding brings the new title into the application.

Welcome to { { title } }!

Import UI-Framework

As this is a web application showing you how to build an Angular app, there is no need to do CSS styling. Probably you already know how to style with CSS and the focus here should be Angular, that's why we are going to use the Semantic UI CSS-Framework for a nice look and feel.

With the plus button you can add another terminal to type in new commands while running the application.

You can install the UI-Framework with the command:

npm i --save semantic-ui-css

Add to the file angular.json:

"styles": [
  "../Where-To-Go/node_modules/semantic-ui-css/semantic.css",
  "src/styles.css"
],

Now you can test the new style. Adjust the app.component.html like this and see if it looks different:

<div style="text-align:center" >
  <h1 class="ui header" style="margin: 30px">
    Welcome to { { title } }!
  </h1>
</div>
<router-outlet> </router-outlet>
