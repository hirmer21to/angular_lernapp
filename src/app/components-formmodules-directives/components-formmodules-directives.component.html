Mock Data

Later we will have more than one lecture and therefor we are going to need a list to display all of them. Because we don't get our data from a server we will create some mock-data and pretend they are from a server.

ng g class shared/mock-lectures

In this class we define the constant LECTURES as an Array of lectures and export it. The content is the same like the property lecture from the lecture-item.component.ts, so you can copy the first from there and paste them a few times for different lectures.

   id: 1,

   department: 'Lehrstuhl f√ºr Medieninformatik'

  {

... and so on
  }

After that, you can delete the property lecture from the lecture-item.component.ts and replace it with the constant.

lectures = LECTURES;

Display a List

Now we generate our shell for the lecture items. The timetable is going to be a list with all the lectures we have.

ng g component timetable

We display this component in the app.component.ts and replace the lecture-item with this code:

<app-timetable></app-timetable>

In the timetable.component.html we display now the list of lecture-items with a header beyond:

<div style="margin-top: 15px"></div>
<h2 class="ui header">Timetable:</h2>
<app-lecture-item></app-lecture-item>

In the lecture-item.component.html we can make some changes now to actually show a list of lecture-items. In the div with class="item" we insert a for-loop.

<div *ngFor="let lecture of lectures" class="item"></div>

Directives

These constructs are called "Directives". With this you can make loops in the HTML similar to loops in the logic code (TypeScript, Java,...), like you may know.

In this code example every time I iterate over my list of lectures I take one single lecture out of all lectures and for this lecture is display the name, the location, the day and time and so on. And with this for-loop I repeat this for all the lectures I have.

Another directive we already used before is *ngIf. Because we made department a optional property, someone may not set a value for this property. So we want to ask if this property has a value and just display the corresponding div if it has.

And never forget the *:
*ngFor="let lecture of lectures"
*ngIf="lecture.department"

Details View

Later we want to be able to make changes within a lecture item so we want select one lecture and get its details view via click. So first we need a variable for the selected lecture in the lecture-item.component.ts and second we need a function, we call it onSelect(), which assigns the clicked lecture from the template to the selectedLecture of the component.

selectedLecture: Lecture;
onSelect(lecture: Lecture): void {
  this.selectedLecture = lecture;
}

We add a click-event-binding to the lecture-item.component.html and change the whole div bock to an active element.

<a *ngFor="let lecture of lectures" (click)="onSelect(lecture)" class="item">

We want to display the clicked lecture beneath the list of all lectures. Therefor we need to add some code for the selected lecture, but just if there is a lecture selected. To make this sure, we have the *ngIf directive.

In the lecture-item.component we need to insert the following code snippet:

<div class="ui middle aligned divided list">
  <div *ngIf="selectedLecture" style="padding: 50px;">
   <h4> { {selectedLecture.name} } Details </div>
   <div class="description">Where? { {selectedLecture.building} }/{
   {selectedLecture.floor} }.{ {selectedLecture.room} } </div>
   <div class="description">When? { {selectedLecture.day} },
   { {selectedLecture.time.hours} }:{ {selectedLecture.time.minutes} } </div>
   <div *ngIf="selectedLecture.department" class="description">Department:
   { {selectedLecture.department} } </div>
  </div>
</div>

Now we asked, is there a selectedLecture, okay then display the selectedLecture.name selectedLecture.building and all the other information.
