Generate Details Component

The lecture details should be reusable, that's why they will have an extra component from now on. Just cut out the HTML code about the details from timetable.component.html. It is the whole ui form. Generate the component and paste it in lecture-details.component.html.

ng g component lecture-details

If we want to use the details component not only for a selected item but also for every lecture item, we need to change selectedLecture to lecture again.

The lecture property in lecture-details.component.ts must be an input property which is commented on by the @Input()-decorator, as it binds the external LectureItemComponent to it.

Insert this right above the empty constructor:

@Input() lecture: Lecture;

This results in making these imports:

import { Component, OnInit, Input } from "@angular/core";
import { Lecture } from "../shared/lecture";

And put this in the timetable.component.hmtl:

<app-lecture-details [lecture]="selectedLecture"></app-lecture-details>

If you have components that have a main and a detail view, you should seperate them for better maintenance. In this expamle with the list of lectures, the component receives a lecture object through its lecture property and indicates it. Now every time the user selects a lecture from the list, it will be displayed in the details component.

With the selector app-lecture-details the component can be displayed.

[lecture]="selectedLecture" creates a one-way-binding from the selectedLecture property of the TimetableComponent to the lecture property of the target element, which is assigned to the lecture property of the LectureDetailsComponent.

If the user clicks on an item of that list, the selectedLecture changes. This updates the lecture because of the property-binding and the LectureDetailsComponent displayes the chosen object.

In this section we only made "inside changes". That means that you can't see any difference in the user interface. We seperated the list and the details-view. Now a development of both components is possible, without any need to change the other one. The details-component is now reusable.
