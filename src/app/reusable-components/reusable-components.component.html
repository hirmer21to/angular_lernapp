Reusable Components

Generate Details Component

The lecture details should be reusable, that's why they will have an extra component from now on. Just cut out the HTML code about the details from timetable.component.html. It is the whole ui form. Generate the component and paste it in lecture-details.component.html.

ng g component lecture-details

If we want to use the details component not just for a selected item but also for every lecture item, we need to change selectedLecture to lecture again.

The event property in lecture-details.component.html must be an input property that is commented on by the @Input() decorator, as it binds the external LectureItemComponent to it.

@Input() lecture: Lecture;

This results in making these imports:

import { Component, OnInit, Input } from '@angular/core';
import { Lecture } from '../shared/lecture';

And put this in the timetable.component.hmtl:

<app-lecture-details [lecture]="selectedLecture"></app-lecture-details>

This component receives a lecture object through its lecture property and indicates that. Now every time the user selects a lecture from the list this lecture will be displayed in the details component.

With the selector app-lecture-details the component can be displayed.

[lecture]="selectedLecture" creates a one-way-binding from the selectedLecture property of the TimetableComponent to the lecture property of the target element, which is assigned to the lecture property of the LectureDetailsComponent.

If the user clicks on an item from that list, the selectedLecture changes, this updates the lecture because of the property-binding and the LectureDetailsComponent displayes the chosen object.

In this section, there were just "inside changes", you cannot see any difference in the user interface. But we seperated the list and the details-view. Now a development of both components is possible, without any need to also change the other one. The details-component is now resusable.
