How to set up the Environment:

The Editor

As a perfect code editor for Angular I would recommend Visual Studio Code. It works for all operating systems and comes with an own Git integration and support for TypesScript.

If you don't have it already, just download it:

Visual Studio Code Download Link

Node.js

Node.js is a runtime environment for running JavaScript on the server and can be used for developing frontend and backend. It comes with the Node Package Manager (NPM).

If you don't have Node, you can download and install it:

Node.js Download Link

Next you should try to get the version number on your command line:

node -v

npm -v

Angular requires Node.js version 8.x or 10.x, you always need to keep it up-to-date.

Now you can install packages like this:

npm install your-package-name

Or global like this:

npm install -g your-package-name

Angular-CLI

Angular-CLI is a command line tool based on Node.js which helps you from the very beginning. When you start a new project, the CLI prepares everything for you: It creates your files, installs the npm-packages and creates Git for you. It automatically generates skeletons for components, services, pipes and directives.

For Windows users I recommend windows-build-tools for a frictionless setup. You need to execute this installation as an administrator.

npm install -g windows-build-tools

Now you can install the Angular-CLI by using this command:

npm install -g @angular/cli

In some cases it might be required to install the Angular-CLI as the user, that will use it. So if it does not work, try to install it with your local user.
