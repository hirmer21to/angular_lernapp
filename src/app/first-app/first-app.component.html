Let's start

To get a simple welcome app, first open your command line and type: 

ng new my-first-app

You will get questions about the features ti include in your app:

  • ? Would you like to add Angular routing?". Answer it with Yes! Therefore you get a router to navigate between states and view within your app.
  • ? Which stylesheet format would you like to use? In this tutorial we use plain CSS. The other options SCSS, Sass, Less and Stylus are other stylesheet languages, with further features like functions and variables, which are compiled to CSS through their CSS preprocessor.

A project which includes all necessary files and folders will be created.

These are:

  • The new workspace with the root folder my-first-app.
  • An initial skeleton app project, also called "my-first-app" which you will find it in the subfolder src.
  • An end-to-end test project in the subfolder e2e.
  • The, by npm installed, packages in the subfolder node_modules.
  • Related configuration files.

After installation you get an answer from your system, like this: added 1098 packages from 1166 contributors and audited 39144 packages in 35.242s found 3 vulnerabilities (2 low, 1 high). npm scanned your whole project with all your packages for vulnerabilities and shows you the summarized results. If there are vulnerabilities just follow the instructions on your command line (npm audit fix or similar) to automatically install any compatible updates to vulnerable dependencies. For production-ready code you should fix at least all high vulnerabilities, but as we only learn for ourselves you can ignore all eventually remaining vulnerabilities.

Then you can navigate to that project-folder:

cd my-first-app
ng serve --open

With serve you can already run the application and --open opens the browser directly on http://localhost:4200/ and shows your brand new application (instead of localhost you can type http://127.0.0.1:4200).
