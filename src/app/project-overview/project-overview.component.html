What am I able to do after this Tutorial?

Probably the most interesting question. Afterwards you should have a solid grasp about the following concepts in Angular:

  • Create an application with reusable components.
  • Show a list of data items by using built-in Angular directives.
  • Use directives to show and hide items.
  • Create, read, update and delete data blocks.
  • Create forms to submit user input.
  • Use one-way and two-way data binding to synchronize views with the backend logic.
  • Bind methods to events, like a button click.
  • Use routing to navigate through components and their views.
  • Format data by using pipes.
  • Use a service to encapsulate your business logic and functionality.

What is this project about?

Every student might know the problem. It's a new semester, you have a lot of lectures and you never know which room to go next. Wouldn't it be nice to have an app that shows you where to go at first sight by just opening it?

User Story

As a student
I want to have a dashboard that shows me the room of my next upcoming lecture,
to see where to go next at first sight.

So this is basically the whole idea. On the start page we will display the next lecture of the day.
We used the format of a user story as it best describes the requirements and wishes from the user's point of view. This approach is common in the software development process to gather the user's expectation apart from any technical detail.
Following from that, we will have to manage the lecture data backlog. It will be possible to view all lectures in a timetable, to create new ones, update existing lectures or delete them.

How should this app look like?

When you start the app you will see the next lecture you have to go to on the "Home-Screen". Above the Home-Screen you can find the navigation bar. There, it is possible to go back to the Home-Screen immediately at all times through clicking > Home.

When you click on > Timetable you will get to the list of lectures you have. Here you can see that every lecture can be deleted (> Delete). You can also delete all lectures with one click (> Delete All). Furthermore you can click the button > Add to create a new lecture entry. With a click on one lecture you will get to the details of this specific lecture and you can change them.

So here you can change some details and save them (> Save). Then you go back to the Timetable and see the updated list of lectures (> Go Back).

The last point in the menu is > Add. From there, or with the button > Add Lecture in the > Timetable view you can get to the > Add view. There you can add a totally new lecture by making some input and choosing options from the drop-down menues.

Once again we used a proofen method from the software development tool box: A wireframe. The wireframe enables us to have a early visual prototype of our conceptual design and to reach a common understanding about the fundamental elements on the web page.

What steps follow to proceed?

You will learn the basic concepts of Angular. For every concept like components, routing, service and so on, you will have an own chapter with theoretical parts (light-grey background) and practical parts (instructions on white background). Every chapter will bring you closer to the introduced app. At the beginning of every chapter you will find a user story (light-blue background) about what you want to achieve and at the end of every chapter you will find the list of things you've learned (light-green background), the visualization of the components and how they are arranged, the current code and how it looks.

The current code is displayed in an embedded online code editor named Stackblitz. At the end of every chapter you can explore the code of the whole project in this online editor.

  1. With "open in a new tab" you can open the Stackblitz editor in a new tab.
  2. This is the tab card of the currently opened class or file in the editor.
  3. With a click on this icon you get the file structure of the project and you can open all other files in the embedded editor.
  4. You can switch the displayed content in the embedded code editor. You can display editor and preview side by side or only the editor or only the preview.
  5. You can search in the whole project. You will get a search bar when clicking on the magnifier icon.

Let me summarize: you can use the embedded editor like every code editor you know. It is based on Visual Studio Code. You can make changes and you will see the results immediately or you can just look through the code. None of your changes are permanent, so play with the code as you like.
